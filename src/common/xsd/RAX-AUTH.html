@unlock
type xsd:boolean
use optional

After a number of unsuccessful multifactor authentication attempts a user's account will become locked. When set to true, and the caller has appropriate permissions, this will unlock the user's account if the user has multifactor enabled. When set to false or not provided, no change will be made to the lock status of the user's account.

BypassCodes

@codes
type RAX-AUTH:StringList
use optional

A list of bypass codes for the user

@validityDuration
type xsd:duration
use optional

An attribute of the RAX-AUTH:bypassCodes element that determines how long the bypass codes remain valid. If no value is specified, the value defaults to 30 minutes PT30M.

If the bypass codes are for your own account, customize the value as needed.

Administrators can configure an interval between 1 and 360 minutes when creating a bypass code for other user accounts.

The value is expressed in xsd:duration format, PT20MPnYnMnDTnHnMnS where

P indicates the period (required)

nY indicates the number of years

nM indicates the number of months

nD indicates the number of days

T indicates the start time of a section (required if the duration value includes hours, minutes, or seconds)

nH indicates the number of hours

nM indicates the number of minutes

nS indicates the number of seconds

@numberOfCodes
type xsd:integer
use optional

The number of codes requested.
